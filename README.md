# Flow Captcha Worker

Flow Captcha Worker 是一款基于 Puppeteer 的自动化浏览器集群管理与验证码解决方案，专为处理流式访问和验证码生成场景设计。项目内置了动态指纹伪装机制，并提供了一个现代化的可视化管理面板，便于大规模调度和监控浏览器实例。

## ✨ 核心特性

- **可视化控制台**：基于 Next.js 与 shadcn/ui 构建的响应式 Dashboard，支持实时监控系统资源（CPU/内存）、活跃实例状态与队列处理进度。
- **纯原生数据库支持**：底层采用原生 Node.js 22+ 的 `node:sqlite` 驱动，无需安装或编译第三方 SQLite 依赖，确保多端和多平台的直接兼容。
- **动态 API 密钥管理**：提供完整的鉴权与分发体系。管理员可从控制台直接派发临时或限额的 API Keys 供业务端调用，并支持详尽的日志溯源追踪。
- **预热沙箱调度池 (Warm Pool)**：系统后台维持一定数量的常驻就绪浏览器实例，避免“一次性冷启动”开销，实现毫秒级的任务分派响应。
- **代理节点路由**：原生支持外部代理节点的录入与订阅配置。系统会自动进行健康检查与在线测速，隔离超时失效节点，保障打码业务的高可用性。

## 🚀 部署指南

### 环境要求
- **Node.js** >= 22.5.0（必需：本项目利用原生 SQLite 驱动）
- **包管理器**：npm

### 快速安装与运行

```bash
# 安装根目录运行依赖与控制台前端依赖
npm install 

# 启动服务端引擎及前端控制台 (开发模式)
npm run dev
```

启动完成后，请使用浏览器访问后台管理面板：
👉 **http://localhost:3000**

> **关于管理员账号**
> 首次启动时，系统将在当前环境自动建立 `data.sqlite` 文件，并自动创建默认管理员账号：
> - 用户名：`admin`
> - 密　码：`admin`
> *(为保证安全，强烈建议您在首次登录后前往设置项或调用接口修改此默认密码)*

### 生产环境部署 (Production)

当需要在线上服务器长期运行时，推荐采用先构建后运行的模式：

```bash
# 预先构建 Next.js 生产态前端数据
npm run build 

# 启动正式的常驻服务组合
npm run start:prod
```

## ⚙️ 业务层接口接入

第三方业务系统或爬虫客户端在向 Flow Captcha Worker 获取验证票据时，仅需将控制台发放的 `API Key` 携带至头部并进行请求：

```bash
curl -X POST http://localhost:3000/api/v1/solve \
  -H "Authorization: Bearer <管理面板签发的 API Key>" \
  -H "Content-Type: application/json" \
  -d '{"action": "IMAGE_GENERATION", "project_id": "test_app_id"}'
```

调用成功后，服务集群会分配可用的 Browser Worker 提取有效数据并即时返回。同时，此次操作的元数据（消耗额度、来源 IP 等）将自动归档至系统后端的打码流转日志中。

## 🔰 板块导览
管理后台主要分为以下四个模块：
- **仪表盘总览**：图表化展示整个服务的运行负载、资源占用曲线及系统可用吞吐量。
- **代理节点群**：录入私人代理 IP 或外部分发链接，维持系统内部与节点的连接心跳检测机制。
- **打码日志**：实时呈现每一个鉴权请求的拦截、放行、处理用时以及状态码明细反馈。
- **实例集群**：展示当前预热驻留或正在执行高并发计算任务的所有独立无头浏览器沙盒状态。
